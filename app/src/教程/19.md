# 📅 Android第19天完整版｜动态权限申请（Android6.0+必备）
## 今日学习目标

2. 掌握`READ_EXTERNAL_STORAGE/WRITE_EXTERNAL_STORAGE`存储权限申请
3. 手写权限工具类，一键申请、判断是否授权

## 一、基础知识点
### 1.1 两种权限
1. **普通权限**：清单注册即可，安装自动授权（网络、震动），不用动态申请
2. **危险权限**：**存储、相机、电话、定位**，APP运行时弹窗手动授权，拒绝无法使用功能

### 1.2 存储权限清单配置
打开 `AndroidManifest.xml`，`<application>` 标签上方添加权限：
```xml
<!-- 外部存储读写（安卓6~12） -->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
<!-- Android13单独图片权限（可选） -->
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES"/>
```

## 二、封装权限工具类 PermissionUtil.kt（通用，全项目复用）
新建工具类，封装「判断权限+申请权限」逻辑
```kotlin
import android.app.Activity
import android.content.pm.PackageManager
import androidx.core.app.ActivityCompat
import androidx.core.content.ContextCompat

class PermissionUtil {
    companion object {
        // 判断单个权限是否已授权
        fun hasPermission(act: Activity, permission: String): Boolean {
            return ContextCompat.checkSelfPermission(act, permission) == PackageManager.PERMISSION_GRANTED
        }

        // 申请权限，requestCode：请求标记码（自定义数字）
        fun requestPerm(act: Activity, perms: Array<String>, code: Int) {
            ActivityCompat.requestPermissions(act, perms, code)
        }
    }
}
```

```kotlin
//权限请求码
private val REQ_STORAGE = 1001
//需要申请的权限数组
private val STORAGE_PERM = arrayOf(
    android.Manifest.permission.READ_EXTERNAL_STORAGE,
    android.Manifest.permission.WRITE_EXTERNAL_STORAGE
)
lateinit var ivShow: ImageView
```
onCreate绑定控件：
```kotlin
ivShow = findViewById(R.id.iv_show)
val btnOpenImg = findViewById<Button>(R.id.btn_open_img)

btnOpenImg.setOnClickListener {
    //先判断有没有权限
    if(PermissionUtil.hasPermission(this,STORAGE_PERM[0])){
        //有权限，打开相册
        openAlbum()
    }else{
        //无权限，发起申请
        PermissionUtil.requestPerm(this,STORAGE_PERM,REQ_STORAGE)
    }
}
```

## 四、重写权限回调方法（用户授权/拒绝后回调）
```kotlin
//权限申请结果回调
override fun onRequestPermissionsResult(
    requestCode: Int,
    permissions: Array<out String>,
    grantResults: IntArray
) {
    super.onRequestPermissionsResult(requestCode, permissions, grantResults)
    if(requestCode == REQ_STORAGE){
        //判断第一个权限是否通过
        if(grantResults.isNotEmpty() && grantResults[0]==PackageManager.PERMISSION_GRANTED){
            openAlbum()
        }else{
            Toast.makeText(this,"需要存储权限才能打开相册",Toast.LENGTH_SHORT).show()
        }
    }
}
```

## 五、打开相册代码（隐式跳转系统相册）
```kotlin
//定义相册请求码
private val ALBUM_CODE = 2001

private fun openAlbum(){
    val intent = android.content.Intent(android.content.Intent.ACTION_PICK)
    intent.type = "image/*"
    startActivityForResult(intent,ALBUM_CODE)
}

//接收相册返回的图片
override fun onActivityResult(requestCode: Int, resultCode: Int, data: android.content.Intent?) {
    super.onActivityResult(requestCode, resultCode, data)
    if(requestCode == ALBUM_CODE && resultCode == RESULT_OK){
        val uri = data?.data
        ivShow.setImageURI(uri)
    }
}
```

## 七、补充知识点
1. **永久拒绝权限**：勾选不再询问后，只能跳应用设置手动开启权限（拓展后续学）
2. Android13：废弃`WRITE_EXTERNAL_STORAGE`，读取图片只用`READ_MEDIA_IMAGES`
3. 常用危险权限：CAMERA相机、ACCESS_FINE_LOCATION定位、CALL_PHONE打电话

## 🎯今日必背三句话
1. **普通权限清单注册即可，危险权限必须动态申请**
2. `checkSelfPermission`查授权状态，`requestPermissions`发起申请
3. `onRequestPermissionsResult`接收用户授权结果
