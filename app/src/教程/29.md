## 一、基础概念
Fragment：依附Activity的模块化页面，不能单独存在，生命周期受宿主Activity管控
- add：添加碎片，原有页面保留在底层不销毁
- replace：替换容器内所有碎片，旧Fragment销毁

## 二、一、静态加载（简单演示，项目少用）
xml直接写fragment标签，固定页面
```xml
<fragment
    android:id="@+id/frag_static"
    android:name="com.xxx.HomeFragment"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"/>
```
缺点：运行后无法动态切换，项目主流用**动态加载**。

## 三、动态加载Fragment（核心Transaction事务）
### 1.事务常用方法
1. `add(容器id,fragment)`：新增碎片，覆盖在上层
2. `replace(容器id,fragment)`：清空容器原有碎片，替换新的
3. `remove(fragment)`：移除碎片
4. `hide/show(frag)`：隐藏/显示，不销毁实例
5. `addToBackStack("tag")`：加入回退栈，返回键可以回退上一页
6. `commit()`：提交事务

### 2.封装切换工具（MainActivity原有切换方法升级，支持回退栈）
```kotlin
//替换
fun replaceFragment(frag: androidx.fragment.app.Fragment, addStack:Boolean = false){
    val ft = supportFragmentManager.beginTransaction()
    ft.replace(R.id.fl_content,frag)
    if(addStack){
        ft.addToBackStack(frag.javaClass.name)
    }
    ft.commit()
}
//add方式
fun addFragment(frag: androidx.fragment.app.Fragment, addStack:Boolean = false){
    val ft = supportFragmentManager.beginTransaction()
    ft.add(R.id.fl_content,frag)
    if(addStack){
        ft.addToBackStack(null)
    }
    ft.commit()
}
```
> 使用：`replaceFragment(homeFrag,false)` 不加入回退栈；true加入回退栈，按返回回到上一个碎片

## 四、Fragment完整生命周期（7个）
依附Activity，Activity销毁则Fragment跟着销毁
```
onAttach→onCreate→onCreateView→onStart→onResume→onPause→onStop→onDestroyView→onDestroy→onDetach
```
- onAttach：和宿主Activity绑定（通信初始化在这里）
- onCreateView：加载布局（inflate布局唯一位置）
- onDestroyView：销毁视图，实例还在内存
- onDetach：和宿主解绑

### 示例代码（HomeFragment）

## 五、Fragment ↔ Activity 通信（接口回调标准写法）
需求：Fragment点击按钮，通知Activity执行逻辑
### 1.定义回调接口
```kotlin
interface FragCallBack{
    fun sendMsg(msg:String)
}
```
### 2.Fragment定义接口变量
```kotlin
private var callback:FragCallBack?=null
override fun onAttach(context: Context) {
    super.onAttach(context)
    if(context is FragCallBack){
        callback = context
    }
}
//按钮点击
btnSend.setOnClickListener {
    callback?.sendMsg("来自首页碎片的数据")
}
```
### 3.MainActivity实现接口
```kotlin
class MainActivity:AppCompatActivity(),FragCallBack{
    override fun sendMsg(msg: String) {
        Toast.makeText(this,"收到：$msg",Toast.LENGTH_SHORT).show()
    }
}
```

## 六、ViewPager2 + Fragment 滑动三页面（首页/用户/我的左右滑动）
### 1.布局activity_main.xml，在FrameLayout位置替换ViewPager2
```xml
<androidx.viewpager2.widget.ViewPager2
    android:id="@+id/view_pager"
    android:layout_width="match_parent"
    android:layout_height="0dp"
    android:layout_weight="1"/>
```
### 2.适配器FragVPAdapter
```kotlin
import androidx.fragment.app.Fragment
import androidx.fragment.app.FragmentActivity
import androidx.viewpager2.adapter.FragmentStateAdapter

class FragVPAdapter(act:FragmentActivity, private val fragList:List<Fragment>):FragmentStateAdapter(act) {
    override fun getItemCount() = fragList.size
    override fun createFragment(position: Int): Fragment = fragList[position]
}
```
### 3.MainActivity绑定ViewPager2，联动底部RadioGroup
```kotlin
//碎片集合
val frags = listOf(homeFrag,userFrag,mineFrag)
val adapter = FragVPAdapter(this,frags)
binding.viewPager.adapter = adapter

//页面滑动修改底部选中
binding.viewPager.registerOnPageChangeCallback(object :ViewPager2.OnPageChangeCallback(){
    override fun onPageSelected(position: Int) {
        super.onPageSelected(position)
        when(position){
            0->binding.rgTab.check(R.id.rb_home)
            1->binding.rgTab.check(R.id.rb_user)
            2->binding.rgTab.check(R.id.rb_mine)
        }
    }
})
//底部按钮切换页面
binding.rgTab.setOnCheckedChangeListener { _,id->
    when(id){
        R.id.rb_home->binding.viewPager.currentItem=0
        R.id.rb_user->binding.viewPager.currentItem=1
        R.id.rb_mine->binding.viewPager.currentItem=2
    }
}
```
