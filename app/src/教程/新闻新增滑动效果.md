## ✅ 新闻页面"滑动到底加载更多"功能已完成

### 改动了4个文件：

### 1️⃣ `ApiConfig.kt` — 新增分页URL支持
- 新增 `NEWS_BASE_URL` 和 `NEWS_PAGE_SIZE` 常量
- 新增 `getNewsUrl(page: Int)` 方法，支持传入页码生成分页URL
- 保持 `NEWS_URL` 向后兼容（默认第1页）

### 2️⃣ `item_footer.xml` — 新建底脚布局
- 包含一个 `ProgressBar`（加载旋转动画）和 `TextView`（提示文字）
- 加载中状态：显示转圈 + "正在加载..."
- 到底状态：隐藏转圈 + "—— 到底了 ——"

### 3️⃣ `NewsAdapter.kt` — 支持多种ViewType
- 使用 `getItemViewType()` 区分普通条目（`TYPE_ITEM`）和底脚（`TYPE_FOOTER`）
- 新增 `FooterVH` 内部类，管理底脚的显示状态
- 新增 `footerState` 字段（`STATE_LOADING` / `STATE_END`）
- 新增 `appendData()` 方法，追加数据而非替换
- 新增 `setFooterState()` 方法，切换底脚显示状态
- `getItemCount()` = 数据条数 + 底脚1条

### 4️⃣ `NewsFragment.kt` — 核心加载更多逻辑
- 新增分页状态变量：`currentPage`（当前页码）、`isLoading`（防重复加载）、`hasMore`（是否还有更多）
- 添加 `RecyclerView.OnScrollListener` 滑动监听
    - 当滑动到**倒数第2个条目**时自动触发加载
    - 只有向下滑动（`dy > 0`）时才触发
- 新增 `loadMoreNews()` 方法：
    - 页码 +1 请求下一页
    - 成功时追加数据到列表
    - 返回数据 < 每页数量时显示"到底了"
    - 返回空列表时显示"到底了"
    - 请求失败时回退页码，允许重试

### 🔄 加载流程：
```
滑动到底 → OnScrollListener检测 → isLoading=false且hasMore=true
    → isLoading=true，currentPage++，请求下一页
    → 成功 → appendData追加数据 → 判断是否还有更多
    → 失败 → currentPage--，恢复状态
```
